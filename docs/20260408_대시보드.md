# Spark Application 메트릭 라벨 구성 - 2026-04-08

## 개요

Spark Operator의 `spark_application_success_execution_time_seconds` 메트릭에 커스텀 라벨을 추가하여 전 구간 모니터링을 활성화합니다.

## 목표

- service_id를 기반으로 전 구간 모니터링 가능
- Spark Application의 처리 시간 분석
- PromQL을 통한 대시보드 구성

## 변경 사항 요약

### 1. 소스 코드 변경

#### 1.1 템플릿 변경 (`template/0002_wfbm.yaml`)

SparkApplication의 metadata.labels에 커스텀 라벨 추가:

```yaml
labels:
  yunikorn.apache.org/app-id: "SERVICE_ID_PLACEHOLDER"
  yunikorn.apache.org/queue: "QUEUE_PLACEHOLDER"
  build-number: "BUILD_NUMBER"
  spark-app: "true"
  # Custom labels for metrics tracking
  app-name: "clustering"
  service-name: "CATEGORY_PLACEHOLDER"
  service-id: "SERVICE_ID_PLACEHOLDER"
  provision_id: "PROVISION_ID_PLACEHOLDER"
  app_type: "ias"
  queue: "QUEUE_PLACEHOLDER"
```

**라벨 설명:**

| 라벨 | 값 | 소스 | 설명 |
|------|-----|------|------|
| `app-name` | `clustering` | 하드코딩 | 애플리케이션 이름 (고정) |
| `app_type` | `ias` | 하드코딩 | 애플리케이션 타입 (고정) |
| `provision_id` | 요청 파라미터 | `provision_id` | 프로비저닝 ID |
| `service-id` | `{service_id}-{category}-{uid}` | 조합 | 고유 서비스 식별자 |
| `service-name` | 요청 파라미터 | `category` | 서비스 명/카테고리 |
| `queue` | 리소스 계산 결과 | MinIO/Config | YuniKorn 큐 |

#### 1.2 서비스 코드 변경 (`services/template.go`)

카테고리 및 프로비저닝 ID 치환 함수 추가:

```go
// ApplyCategoryToYAML - YAML의 CATEGORY_PLACEHOLDER를 실제 category 값으로 치환
func ApplyCategoryToYAML(yamlStr string, category string) string {
	return strings.ReplaceAll(yamlStr, "CATEGORY_PLACEHOLDER", category)
}

// ApplyProvisionIDToYAML - YAML의 PROVISION_ID_PLACEHOLDER를 실제 provision_id 값으로 치환
func ApplyProvisionIDToYAML(yamlStr string, provisionID string) string {
	return strings.ReplaceAll(yamlStr, "PROVISION_ID_PLACEHOLDER", provisionID)
}
```

#### 1.3 핸들러 변경 (`handlers/create.go`)

`handleEnabledProvision` 함수에서 치환 함수 호출:

```go
// Category 라벨 적용 (메트릭용)
yamlTemplate = services.ApplyCategoryToYAML(yamlTemplate, req.Category)

// Provision ID 라벨 적용 (메트릭용)
yamlTemplate = services.ApplyProvisionIDToYAML(yamlTemplate, req.ProvisionID)
```

### 2. Spark Operator 구성 변경

#### 2.1 메트릭 라벨 활성화

Spark Operator Deployment의 `--metrics-labels` 인자에 커스텀 라벨 추가:

```bash
kubectl patch deployment spark-operator-controller -n default --type=json -p='[
  {
    "op": "replace",
    "path": "/spec/template/spec/containers/0/args/10",
    "value": "--metrics-labels=app_type,app-name,service-name,service-id,queue,provision_id"
  }
]'
```

**변경 전:**
```bash
--metrics-labels=app_type
```

**변경 후:**
```bash
--metrics-labels=app_type,app-name,service-name,service-id,queue,provision_id
```

#### 2.2 확인 방법

```bash
# Spark Operator 파드 확인
kubectl get pods -n default -l app.kubernetes.io/component=controller

# 설정 확인
kubectl exec -n <pod-name> -- cat /proc/1/cmdline | tr '\0' '\n' | grep metrics-labels
```

## 문제 해결

### 7.1 라벨이 메트릭에 나타나지 않을 때

**원인:** Spark Operator의 `--metrics-labels`에 라벨이 포함되지 않음

**해결:**
```bash
# 현재 설정 확인
kubectl get deployment spark-operator-controller -n default -o jsonpath='{.spec.template.spec.containers[0].args}' | jq -r '.[]' | grep metrics-labels

# 설정 수정
kubectl patch deployment spark-operator-controller -n default --type=json -p='[
  {
    "op": "replace",
    "path": "/spec/template/spec/containers[0/args/10",
    "value": "--metrics-labels=app_type,app-name,service-name,service-id,queue,provision_id"
  }
]'

# 롤아웃 대기
kubectl rollout status deployment/spark-operator-controller -n default
```

### 7.2 라벨 값이 PLACEHOLDER로 나올 때

**원인:** hynix 서비스에서 치환하지 않음

**해결:**
1. hynix 서비스가 최신 버전인지 확인
2. 템플릿 파일 확인
3. 로그 확인

```bash
# 로그 확인
tail -f /tmp/hynix.log

# hynix 서비스 재시작
pkill -f hynix-service
./hynix-service > /tmp/hynix.log 2>&1 &
```

### 7.3 SparkApplication이 완료되지 않을 때

**확인 사항:**
1. YuniKorn 스케줄러 상태
2. 리소스 가용성
3. Pod 상태

```bash
# YuniKorn 상태 확인
kubectl get cm yunikorn-config -n default -o yaml

# Pod 상태 확인
kubectl get pods -n default | grep spark

# Pod 로그 확인
kubectl logs <pod-name> -n default
```
